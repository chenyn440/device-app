# UI_GUIDE

## Summary

当前 UI 以 Windows 工业上位机风格为主，采用：

- 顶部菜单
- 左侧导航
- 右侧主工作区
- 页面内头部按钮区
- 谱图区 / 参数区 / 状态区组合布局

当前重点页面为：

- 调谐
- 监测
- 数据
- 设置

## 主窗口结构

### 顶部

- 第一层：`文件 / 运行 / 设置 / 视图`
- 第二层：页面头部操作按钮区

### 左侧导航

当前工作区：

- 调谐
- 监测
- 方法
- 数据
- 设置

## 调谐页

布局特征：

- 顶部操作按钮区
- 头部配置摘要区
- 中部主谱图
- 下部辅助谱图
- 右侧 Tab：参数配置 / 仪器状态 / 设备连接

当前调谐页头部配置区包含：

- 扫描方式
- 起始质量数 / 终止质量数
- 检测器
- 稳定区
- 摘要表

UI 约束：

- 布局结构近期不再大改，优先微调样式
- 头部配置区允许压缩高度，但不改三栏结构
- 右侧参数区继续沿用截图驱动微调

## 监测页

布局特征：

- 顶部方法按钮区
- 左侧 `RIC / TIC`
- 右侧 Tab：参数配置 / 仪器状态

参数区当前包含：

- 检测器
- 稳定区
- 扫描方式
- `SIM` 扫描表格

UI 约束：

- 不改整体布局
- 重点优化右侧参数配置与仪器状态样式

## 数据页

职责：

- 展示单帧数据和本地加载结果
- 支撑后续数据管理与浏览扩展

## 设置页

布局特征：

- 左侧连接与存储配置
- 右侧管道服务测试

当前约束：

- 顶部“本地连接 / 远程中转服务”已改为自定义切换条
- 视觉要求偏紧凑，减少不必要边框和留白

## 当前 UI 工作方式

当前项目 UI 迭代方式是“截图对标 + 局部精修”，因此：

- 先保证结构正确
- 再处理留白、边框、字号、按钮尺寸
- 尽量避免在同一轮同时改布局和样式

## 当前 UI 已知问题

- 页面之间样式常量还未完全统一
- 设计系统还没有抽象成统一主题文件
- 一些页面仍存在固定尺寸较多的问题
- 不同页面的按钮与表格风格仍有细微差异
